## 二、核心模块实现逻辑详解

### 2.1 模块一：高频传感器采集

#### 工作原理

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     传感器采样流程                                        │
│                                                                           │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐                               │
│  │加速度计 │  │ 陀螺仪  │  │ 磁力计  │                               │
│  │200次/秒 │  │200次/秒 │  │50次/秒  │                               │
│  └─────┬────┘  └─────┬────┘  └─────┬────┘                               │
│        │             │             │                                     │
│        ▼             ▼             ▼                                     │
│  ┌─────────────────────────────────────────┐                             │
│  │         数据合并与同步                   │                             │
│  │  - 保存每个传感器的最新读数             │                             │
│  │  - 当加速度计和陀螺仪都更新时才输出      │                             │
│  │  - 磁力计更新慢，用上次的值填空          │                             │
│  └─────────────────┬───────────────────────┘                             │
│                    │                                                      │
│                    ▼                                                      │
│         ┌───────────────────┐                                             │
│         │  IMU数据输出(200Hz)│                                             │
│         └───────────────────┘                                             │
└─────────────────────────────────────────────────────────────────────────┘
```

#### 关键要点
- **加速度计**：测量手机受到的加速度（包括重力），200次/秒
- **陀螺仪**：测量手机旋转的角速度，200次/秒
- **磁力计**：测量磁场方向（类似指南针），50次/秒
- **GPS**：提供绝对位置、速度、方向，1-5次/秒

### 2.2 模块二：震动滤波（解决卡丁车震动问题）

#### 问题分析
卡丁车发动机震动和赛道颠簸会产生高频噪声，让加速度计数据"乱跳"

#### 解决思路：三级滤波组合 + 震动强度自适应

```
┌─────────────────────────────────────────────────────────────────────────┐
│                   震动滤波处理流程                                        │
│                                                                           │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  第一步：震动强度检测                                             │   │
│  │  - 保存最近50个加速度计读数                                       │   │
│  │  - 计算加速度的方差（波动越大方差越大）                           │   │
│  │  - 方差 = 震动强度（弱/中/强三档）                                │   │
│  └─────────────────────────────┬─────────────────────────────────────┘   │
│                                │                                          │
│                                ▼                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  第二步：根据震动强度调整滤波强度                                 │   │
│  │  - 弱震动：轻滤波（保留真实运动细节）                             │   │
│  │  - 中震动：中等滤波                                               │   │
│  │  - 强震动：强滤波（把震动压下去）                                 │   │
│  └─────────────────────────────┬─────────────────────────────────────┘   │
│                                │                                          │
│                                ▼                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  第三步：三级滤波流水线处理                                       │   │
│  │                                                                 │   │
│  │  原始数据 → [低通滤波] → [中值滤波] → [卡尔曼滤波] → 输出         │   │
│  │   (有噪声)    (去高频)    (去尖峰)    (平滑)      (干净)         │   │
│  │                                                                 │   │
│  │  ┌─────────────────────────────────────────────────────────┐   │   │
│  │  │  1. 低通滤波器 (截止频率15Hz)：                            │   │   │
│  │  │     - 允许15Hz以下的信号通过                               │   │   │
│  │  │     - 卡丁车震动主要在20Hz以上，被过滤掉                    │   │   │
│  │  │     - 真实运动频率低，保留下来                             │   │   │
│  │  └─────────────────────────────────────────────────────────┘   │   │
│  │                             │                                    │   │
│  │  ┌─────────────────────────────────────────────────────────┐   │   │
│  │  │  2. 中值滤波器 (窗口大小5)：                               │   │   │
│  │  │     - 取最近5个点的中间值                                  │   │   │
│  │  │     - 把偶尔跳变的"尖峰"去掉                                │   │   │
│  │  └─────────────────────────────────────────────────────────┘   │   │
│  │                             │                                    │   │
│  │  ┌─────────────────────────────────────────────────────────┐   │   │
│  │  │  3. 卡尔曼滤波器：                                         │   │   │
│  │  │     - 预测 + 修正的递归算法                                │   │   │
│  │  │     - 相信运动的连续性，平滑最后的波动                     │   │   │
│  │  └─────────────────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                │                                          │
│                                ▼                                          │
│                     ┌───────────────────┐                                  │
│                     │  干净的加速度数据  │                                  │
│                     └───────────────────┘                                  │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.3 模块三：姿态跟踪与坐标系转换（解决裤兜里手机朝向问题）

#### 核心问题
手机在裤兜里可以随意旋转、倾斜，我们需要：
1. 实时知道手机当前的朝向（姿态）
2. 把所有传感器数据从"手机坐标系"转到"地面坐标系"

#### 解决思路：Madgwick 滤波 + 四元数姿态表示

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      姿态跟踪与坐标系转换流程                                  │
│                                                                               │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  什么是"姿态"？                                                       │   │
│  │                                                                       │   │
│  │  用三个角度描述手机在空间中的旋转：                                    │   │
│  │   - 横滚角 (Roll)：手机绕自己竖轴旋转（类似翻身）                     │   │
│  │   - 俯仰角 (Pitch)：手机绕自己横轴旋转（类似点头）                   │   │
│  │   - 航向角 (Yaw)：手机绕自己纵轴旋转（类似转头，指南针方向）         │   │
│  │                                                                       │   │
│  │  用"四元数"表示这三个角（比直接用角度更稳定，不会有"万向锁"问题）     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                      │                                       │
│  ┌───────────────────────────────────▼───────────────────────────────────┐   │
│  │  Madgwick 滤波的工作原理                                               │   │
│  │                                                                         │   │
│  │  两个数据源相互修正：                                                   │   │
│  │                                                                         │   │
│  │  ┌──────────────────────────┐        ┌──────────────────────────┐   │   │
│  │  │  陀螺仪积分（预测）      │        │  重力+磁力计（修正）     │   │
│  │  │  - 测角速度             │        │  - 重力指向地面           │   │
│  │  │  - 积分算姿态变化       │        │  - 磁力计指北             │   │
│  │  │  - 高频准确（200Hz）    │        │  - 绝对方向（有噪声）     │   │
│  │  │  - 但有累积漂移         │        │  - 低频修正漂移          │   │
│  │  └──────────────────────────┘        └──────────────────────────┘   │   │
│  │           │                                      │                     │   │
│  │           └──────────────────┬───────────────────┘                     │   │
│  │                              ▼                                         │   │
│  │               ┌───────────────────────────┐                            │   │
│  │               │  融合输出：高精度姿态     │                            │   │
│  │               └───────────────────────────┘                            │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                      │                                       │
│  ┌───────────────────────────────────▼───────────────────────────────────┐   │
│  │  坐标系转换（手机 → 地面）                                              │   │
│  │                                                                         │   │
│  │  问题：加速度计测的是相对于手机的加速度，但我们需要相对于地面的         │   │
│  │                                                                         │   │
│  │  解决：用"旋转矩阵"进行转换                                             │   │
│  │                                                                         │   │
│  │  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  │  例子：                                                          │   │
│  │  │  - 手机在裤兜里颠倒了                                           │   │
│  │  │  - 手机坐标系中，重力指向"下"（-Z轴）                            │   │
│  │  │  - 但在地面坐标系中，重力还是指向地面                             │   │
│  │  │  - 旋转矩阵自动把这个颠倒"正过来"                               │   │
│  │  └─────────────────────────────────────────────────────────────────┘   │
│  │                                                                         │   │
│  │  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  │  具体步骤：                                                      │   │
│  │  │  1. 从四元数算出 3x3 旋转矩阵                                   │   │
│  │  │  2. 旋转矩阵 × 手机坐标系向量 = 地面坐标系向量                  │   │
│  │  │  3. 加速度、角速度都做同样的转换                               │   │
│  │  └─────────────────────────────────────────────────────────────────┘   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                      │                                       │
│                                      ▼                                       │
│                          ┌───────────────────────┐                           │
│                          │  所有数据统一到地面！   │                           │
│                          │  手机怎么晃都不怕了！   │                           │
│                          └───────────────────────┘                           │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.4 模块四：重力分离与线性加速度提取

#### 问题
加速度计测的是"总加速度"，包含两部分：
- 重力加速度（恒定向下，9.8m/s²）
- 线性加速度（卡丁车实际加减速的加速度）

我们需要把它们分开，因为只有线性加速度是用来算轨迹的！

#### 解决思路

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      重力分离流程                                          │
│                                                                           │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  输入：总加速度（地面坐标系中）                                   │   │
│  │  = 重力 + 卡丁车运动产生的加速度                                  │   │
│  └─────────────────────────────┬─────────────────────────────────────┘   │
│                                │                                          │
│                                ▼                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  低通滤波器（截止频率0.5Hz，非常慢）                             │   │
│  │                                                                 │   │
│  │  为什么用超慢的低通？                                             │   │
│  │  - 重力的方向是恒定的，几乎不变                                   │   │
│  │  - 卡丁车的加减速是高频变化                                       │   │
│  │  - 超低通只保留恒定的部分 → 这就是重力！                          │   │
│  └─────────────────────────────┬─────────────────────────────────────┘   │
│                                │                                          │
│                                ▼                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  简单减法：                                                       │   │
│  │  线性加速度 = 总加速度 - 重力                                     │   │
│  └─────────────────────────────┬─────────────────────────────────────┘   │
│                                │                                          │
│                                ▼                                          │
│                     ┌─────────────────────┐                              │
│                     │  输出：线性加速度！  │                              │
│                     │  这才是卡丁车实际   │                              │
│                     │  运动产生的加速度！  │                              │
│                     └─────────────────────┘                              │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.5 模块五：扩展卡尔曼滤波（EKF）传感器融合

#### 核心问题
- GPS 精度高但更新慢（1-5Hz）
- IMU 更新快但有漂移（200Hz，但久了位置会偏）

#### 解决思路：EKF 融合两者的优点

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      EKF 传感器融合流程（核心！）                              │
│                                                                               │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  EKF 的基本思想：预测 + 修正的循环                                    │   │
│  │                                                                         │   │
│  │  ┌──────────────────────────┐              ┌──────────────────────────┐   │
│  │  │   预测步骤（200Hz）     │              │   修正步骤（GPS来的时候）│   │
│  │  │  用IMU数据              │              │  用GPS数据               │   │
│  │  │  积分运动方程          │              │  修正预测的漂移         │   │
│  │  │  算新的位置、速度      │              │  GPS越准，修正越多      │   │
│  │  └──────────────────────────┘              └──────────────────────────┘   │
│  │           │                                          │                  │   │
│  │           └───────────────────────┬──────────────────┘                  │   │
│  │                                   ▼                                      │   │
│  │              ┌───────────────────────────────┐                           │   │
│  │              │  输出：高精度的位置和速度      │                           │   │
│  │              │  (200Hz输出，分米级精度)      │                           │   │
│  │              └───────────────────────────────┘                           │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                      │                                       │
│  ┌───────────────────────────────────▼───────────────────────────────────┐   │
│  │  EKF 状态向量（11个数据，同时估计）                                      │   │
│  │                                                                         │   │
│  │  1. 纬度（当前位置）                                                     │   │
│  │  2. 经度（当前位置）                                                     │   │
│  │  3. 高度（当前位置）                                                     │   │
│  │  4. 东向速度（向东还是向西，多快）                                       │   │
│  │  5. 北向速度（向北还是向南，多快）                                       │   │
│  │  6. 垂向速度（向上还是向下，多快）                                       │   │
│  │  7. 横滚角（姿态）                                                       │   │
│  │  8. 俯仰角（姿态）                                                       │   │
│  │  9. 航向角（姿态）                                                       │   │
│  │  10. 加速度计零偏（传感器误差，在线估计）                                │   │
│  │  11. 陀螺仪零偏（传感器误差，在线估计）                                 │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                      │                                       │
│  ┌───────────────────────────────────▼───────────────────────────────────┐   │
│  │  具体工作循环（每秒200次）                                               │   │
│  │                                                                         │   │
│  │  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  │  每个 IMU 数据进来（200Hz）：                                    │   │
│  │  │  1. 线性加速度积分一次 → 速度更新                                │   │
│  │  │  2. 速度再积分一次 → 位置更新                                    │   │
│  │  │  3. 角速度积分 → 姿态更新                                        │   │
│  │  │  （这就是预测步骤，只靠IMU）                                     │   │
│  │  └─────────────────────────────────────────────────────────────────┘   │
│  │                                                                         │   │
│  │  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  │  当 GPS 数据进来（1-5Hz，慢得多）：                              │   │
│  │  │  1. 对比 EKF 预测的位置 vs GPS 测量的位置                        │   │
│  │  │  2. 计算两者的差距（误差）                                       │   │
│  │  │  3. 用"卡尔曼增益"把误差反馈回去修正                             │   │
│  │  │     - GPS 精度高（误差小）→ 增益大，大幅修正                      │   │
│  │  │     - GPS 精度低（误差大）→ 增益小，小幅修正                      │   │
│  │  │  4. 同时修正速度、姿态、零偏（所有状态一起调）                    │   │
│  │  └─────────────────────────────────────────────────────────────────┘   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                      │                                       │
│                                      ▼                                       │
│                          ┌───────────────────────┐                           │
│                          │  最终效果：            │                           │
│                          │  - 200Hz 高频输出     │                           │
│                          │  - 分米级定位精度     │                           │
│                          │  - 无累积漂移         │                           │
│                          └───────────────────────┘                           │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.6 模块六：数据存储（解决200Hz数据量大的问题）

#### 问题
200Hz意味着每秒写200次文件，如果每次都直接写硬盘，会：
1. 速度慢，可能跟不上
2. 频繁写操作伤硬盘

#### 解决思路：环形缓冲 + 批量写入

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      数据存储流程                                          │
│                                                                           │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  第一步：环形缓冲队列（在内存里）                                 │   │
│  │                                                                 │   │
│  │  结构：一个固定大小的数组（比如能存10000个点）                   │   │
│  │  - 数据进来：从队尾加进去                                        │   │
│  │  - 满了就把队头的旧数据顶出去（不过我们会定期存盘）              │   │
│  │                                                                 │   │
│  │  优点：内存读写极快，200Hz完全没问题                            │   │
│  └─────────────────────────────┬─────────────────────────────────────┘   │
│                                │                                          │
│                                ▼                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  第二步：批量写入判断                                          │   │
│  │                                                                 │   │
│  │  检查缓冲队列里有多少点：                                       │   │
│  │  - < 1000个点：继续往内存加                                     │   │
│  │  - >= 1000个点：触发一次批量写入                              │   │
│  └─────────────────────────────┬─────────────────────────────────────┘   │
│                                │                                          │
│                                ▼                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  第三步：批量 JSON 写入硬盘                                    │   │
│  │                                                                 │   │
│  │  文件格式：每行一个 JSON 对象（方便后续读取）                  │   │
│  │  {                                                             │   │
│  │    "timestamp": 1234567890.123,  // 时间戳                     │   │
│  │    "lat": 30.123456789,         // 纬度                        │   │
│  │    "lon": 120.123456789,        // 经度                        │   │
│  │    "alt": 50.5,                 // 高度                        │   │
│  │    "speed": 30.2,               // 速度(km/h)                 │   │
│  │    "accuracy": 0.5,             // 估计精度                    │   │
│  │    "roll": 0.1,                 // 横滚角                      │   │
│  │    "pitch": 0.2,                // 俯仰角                      │   │
│  │    "yaw": 180.0                 // 航向角                      │   │
│  │  }                                                             │   │
│  │  ... 下一个点 ...                                               │   │
│  │                                                                 │   │
│  │  为什么1000个点写一次？                                         │   │
│  │  - 1000个点 = 5秒数据（200Hz）                                  │   │
│  │  - 既保证写的次数不多，又保证断电最多丢5秒数据                  │   │
│  └─────────────────────────────┬─────────────────────────────────────┘   │
│                                │                                          │
│                                ▼                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  停止时的处理：                                                 │   │
│  │  - 把缓冲队列里剩下的所有点都写入文件                          │   │
│  │  - 确保不丢数据                                                │   │
│  └─────────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────────┘
```

## 三、完整数据流程总结

### 从传感器到轨迹文件的完整链路

```
时间起点 ──────────────────────────────────────────────────────────────────>

t0: 启动，静止几秒校准初始姿态
    └─> 重力向量确定初始横滚+俯仰
    └─> 等待开始运动，GPS给出初始航向

t1-tn: 卡丁车运动中（每秒200次循环）

    【每个 IMU 数据点 (200Hz)】
    │
    ├─> 1. 震动滤波处理加速度
    │       └─> 检测震动强度
    │       └─> 三级滤波流水线
    │
    ├─> 2. 更新姿态（Madgwick）
    │       └─> 陀螺仪积分预测
    │       └─> 重力+磁力计修正
    │
    ├─> 3. 坐标系转换（手机→地面）
    │       └─> 加速度转到地面
    │       └─> 角速度转到地面
    │
    ├─> 4. 重力分离
    │       └─> 总加速度 - 重力 = 线性加速度
    │
    ├─> 5. EKF 预测
    │       └─> 积分算新位置、速度
    │       └─> 更新姿态
    │
    ├─> 6. 输出轨迹点
    │
    └─> 7. 加入缓冲队列
            └─> 凑够1000点 → 批量写入JSON文件

    【每个 GPS 数据点 (1-5Hz)】
    │
    └─> 触发 EKF 修正
            └─> 对比预测 vs 测量
            └─> 计算卡尔曼增益
            └─> 修正所有状态（位置、速度、姿态、零偏）

停止：把缓冲队列剩余数据写入，完成！
```

## 四、关键问题解决方案再次回顾

| 问题 | 解决逻辑 |
|------|---------|
| **裤兜里手机任意朝向** | Madgwick滤波实时跟踪3D姿态，用旋转矩阵统一转到地面坐标系 |
| **卡丁车震动干扰** | 震动强度检测 + 三级滤波（低通去高频+中值去尖峰+卡尔曼平滑）+ 自适应滤波强度 |
| **GPS频率低（1-5Hz）** | EKF融合：IMU高频预测（200Hz），GPS低频修正漂移 |
| **数据量大（200Hz）** | 环形内存缓冲 + 批量JSON写入（每1000点/5秒写一次） |
| **分米级精度** | 多传感器融合 + 在线估计传感器零偏 + 运动学约束 |

## 五、测试验证步骤建议

### 阶段一：静态测试（桌子上不动）
1. 手机放桌子上，启动APP
2. 观察姿态输出：应该稳定不变
3. 旋转手机：姿态应该跟着变，停止后稳定
4. 线性加速度输出：应该接近0

### 阶段二：手持步行测试
1. 手持手机，沿着直线走
2. 记录轨迹
3. 用Python读取JSON，画图看轨迹是否直
4. 对比GPS轨迹看精度

### 阶段三：卡丁车实车测试
1. 手机放裤兜（或支架）
2. 跑几圈赛道
3. 分析轨迹数据
4. 对比真实赛道看精度
